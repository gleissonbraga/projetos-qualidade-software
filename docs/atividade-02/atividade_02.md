# Diagnóstico de Qualidade – Startup Local Eats

## 📌 Contexto

A startup **Local Eats** está desenvolvendo uma plataforma digital que conecta clientes a restaurantes locais.

### ⚠ Problemas identificados

- Erros ao finalizar pedidos  
- Pedidos duplicados para restaurantes  
- Funcionalidades com defeitos em produção  
- Falta de clareza sobre responsabilidade da qualidade  

---

## 🎯 Objetivo

Realizar um diagnóstico da qualidade e propor melhorias envolvendo:

- Papéis da equipe  
- Responsabilidades  
- Práticas de QA  
- Perfis profissionais  

---

## 1️⃣ Diagnóstico da Situação Atual

O principal problema da empresa é a **falta de clareza nas responsabilidades**.

Atualmente:

- Todos fazem um pouco de tudo  
- Não existe definição clara de papéis  
- Não há um responsável direto pela qualidade final  

👉 Isso resulta em:

- Falta de validação adequada  
- Entregas com defeitos  
- Baixa confiabilidade do sistema  

👉 Conclusão:  
A qualidade deve ser uma **responsabilidade compartilhada por toda a equipe**, e não apenas de uma única pessoa.

---

## 2️⃣ Papéis da Equipe

### 👨‍💻 Desenvolvedor

**Responsabilidades:**

- Desenvolver funcionalidades  
- Criar testes unitários  
- Garantir qualidade do código  

**Relação com qualidade:**

- Evita defeitos desde a origem (Shift-Left)  

---

### 🧪 QA / Analista de Qualidade

**Responsabilidades:**

- Planejar testes  
- Validar critérios de aceite  
- Identificar e registrar defeitos  

**Relação com qualidade:**

- Garante que o sistema atende aos requisitos  

---

### 📊 Analista de Sistemas

**Responsabilidades:**

- Levantar e revisar requisitos  
- Garantir clareza das regras de negócio  

**Relação com qualidade:**

- Evita erros antes do desenvolvimento  

---

## 3️⃣ Responsabilidades relacionadas à qualidade

| Atividade                         | Responsável              |
|----------------------------------|--------------------------|
| Testar funcionalidades           | QA                       |
| Registrar defeitos               | QA                       |
| Revisar requisitos               | Analista de Sistemas     |
| Testes unitários                 | Desenvolvedor            |
| Validar antes da entrega         | QA + Time                |

---

## 4️⃣ Práticas de QA recomendadas

1. **Shift-Left Testing**  
   - Testar desde o início do desenvolvimento  

2. **Pirâmide de Testes**  
   - Mais testes unitários, menos testes manuais  

3. **Automação de Testes (CI/CD)**  
   - Execução automática de testes  

4. **Testes de Regressão**  
   - Garantir que mudanças não quebrem o sistema  

5. **BDD (Behavior Driven Development)**  
   - Foco no comportamento do usuário

## 5️⃣ Anúncios de Contratação

---

### 💼 Vaga 1: Desenvolvedor(a) de Software

**Empresa:** Local Eats  
**Local:** Porto Alegre – RS  
**Modelo:** Presencial / Híbrido  

#### Sobre a vaga

A Local Eats busca uma pessoa desenvolvedora para criar e manter funcionalidades da plataforma de pedidos online.

#### Principais responsabilidades

- Desenvolver APIs e serviços  
- Criar interfaces responsivas  
- Realizar code reviews  
- Integrar com pagamentos e logística  

#### Requisitos obrigatórios

- Lógica de programação  
- Experiência com JS, TS, Python ou Java  
- Conhecimento em banco de dados  

#### Requisitos desejáveis

- AWS (S3, Lambda, EC2)  
- React ou Node.js  
- Git  
- CI/CD  

---

### 💼 Vaga 2: Analista de Sistemas

**Empresa:** Local Eats  
**Local:** Porto Alegre – RS  
**Modelo:** Presencial / Híbrido  

#### Sobre a vaga

Responsável por conectar necessidades de negócio com soluções técnicas.

#### Principais responsabilidades

- Levantar requisitos  
- Modelar processos  
- Avaliar viabilidade técnica  
- Acompanhar desenvolvimento  

#### Requisitos obrigatórios

- Formação em ADS ou áreas correlatas  
- Escrita de requisitos  
- Conhecimento em SQL  

#### Requisitos desejáveis

- Experiência com e-commerce/delivery  
- Metodologias ágeis  
- Cloud
