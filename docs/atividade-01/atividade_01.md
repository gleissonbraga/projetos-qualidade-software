# Problema Norteador – Projeto Local Eats

## 📌 Visão Geral do Sistema

### 🎯 Propósito

Divulgar o comércio local e estabelecimentos gastronômicos da cidade, conectando usuários a restaurantes.

---

### 👥 Usuários

- Moradores da cidade  
- Turistas  

---

## ⚠ Problemas Prioritários

- Lentidão em horários de pico  
- Resultados incorretos nas buscas  
- Dificuldade em concluir ações  
- Avaliações desaparecendo  
- Falhas em dispositivos móveis  
- Inconsistência entre web e mobile  

---

## 🔍 Análise do Sistema

### 📱 Usabilidade

- Algumas telas são confusas  
- Interface pouco intuitiva  
- Usuário tem dificuldade em navegar  

---

### ⚙ Estabilidade

- Sistema instável em alto volume de acessos  
- Falhas em alguns modelos de smartphone  
- Usuários não conseguem concluir ações  

---

### 🚀 Desempenho

- Tempo de resposta elevado  
- Sistema não escala bem  

---

### 🔄 Compatibilidade

- Diferença entre versão web e mobile  
- Problemas em determinados dispositivos  

---

### 😤 Experiência do Usuário

- Gera frustração  
- Aumenta abandono da plataforma  
- Reduz confiança no sistema  

---

## 🧩 Problemas e Atributos de Qualidade (ISO 25010)

### 🔴 Lentidão em horários de pico

**Atributo:** Performance / Eficiência  
**Justificativa técnica:**

- Alto tempo de resposta  
- Gargalo no banco de dados  
- Falta de cache  
- Ausência de load balancer  
- Baixa escalabilidade do backend  

---

### 🟠 Telas confusas

**Atributo:** Usabilidade  
**Justificativa técnica:**

- Problemas de UX/UI  
- Dificuldade de navegação  
- Usuário perde interesse  

---

### 🔴 Resultados incorretos nas buscas

**Atributo:** Adequação Funcional  
**Justificativa técnica:**

- Problemas em queries  
- Falhas na lógica do backend  
- Perda de credibilidade do sistema  

---

### 🟡 Falhas em smartphones

**Atributo:** Compatibilidade / Portabilidade  
**Justificativa técnica:**

- Problemas com versões de sistema operacional  
- Incompatibilidade com SDKs/bibliotecas  

---

### 🟠 Dificuldade em ações simples

**Atributo:** Usabilidade  
**Justificativa técnica:**

- UX ruim  
- Aumento da taxa de desistência  

---

### 🔴 Avaliações desaparecem

**Atributo:** Confiabilidade / Segurança  
**Justificativa técnica:**

- Problemas de persistência de dados  
- Falha na comunicação entre front-end e back-end  

---

### 🟡 Inconsistência entre web e mobile

**Atributo:** Compatibilidade / Manutenibilidade  
**Justificativa técnica:**

- Diferença entre versões  
- Falta de padronização  
- Design inconsistente  

---

## 📊 Conclusão

Os principais impactos no sistema são:

- Baixa performance  
- Problemas de usabilidade  
- Falta de confiabilidade  

👉 Prioridade de correção:

1. Performance (lentidão)  
2. Busca incorreta  
3. Finalização de ações  
4. Persistência de dados  

Esses pontos impactam diretamente a experiência do usuário e a reputação da plataforma.
